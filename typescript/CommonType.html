    # 常见类型

    # 原始类型：string、number、boolean、bigint、symbol

    # 数组：Array

    # any

    当你不希望一个值导致类型检查错误的时候，就可以设置为 any

    # noImplicitAny

    如果你没有指定一个类型，TypeScript 也不能从上下文推断出它的类型，编译器就会默认设置为 any 类型。
    阻止这种行为，使用noImplicitAny (opens new window)编译选项，当被隐式推断为 any 时，TypeScript 就会报错。

    tsc --noImplicitAny 文件名

    # 函数

    # 参数类型注解

    当你声明一个函数的时候，你可以在每个参数后面添加一个类型注解，声明函数可以接受什么类型的参数。参数类型注解跟在参数名字后面：

    function add(a: number, b: number) {
  return a + b
}

    类型传入错误时会报错

    # 返回值类型注解

    function add(a, b): number {
  return a + b
}

    # 接口

    接口声明（interface declaration）是命名对象类型的另一种方式：

    interface myType {
  x: number;
  y: number;
}

    接口和类型别名非常相似，大部分时候，你可以任意选择使用。接口的几乎所有特性都可以在 type 中使用，两者最关键的差别在于类型别名本身无法添加新的属性，而接口是可以扩展的。

    # 通过继承扩展

    interface Animal {
  name: string
}

interface Bear extends Animal {
  honey: boolean
}

    # 通过交集扩展

    type Animal = {
  name: string
}

type Bear = Animal & { 
  honey: boolean 
}

    # 对一个已经存在的接口添加新的字段

    interface Window {
  title: string
}

interface Window {
  ts: TypeScriptAPI
}

    # 类型断言

    # 断言的两种方式

    • as语法
    interface Person{
  name:string;
  age: number
}

let student = {} as Person
    • <>语法
    interface Person{
  name:string;
  age: number
}

let student = <Person>{}

    # 非空断言操作符（后缀！）

    function log(x?: number | null) {
  // No error（没有！则会报错：Object is possibly 'null'）
  console.log(x!.toFixed());
}

    # 小贴士

    • 慎用类型断言，除非你真的有把握
    • 类型断言是一个编译时语法，不涉及运行时
